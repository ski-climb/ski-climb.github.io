First Open Source Contribution

17 April 2017

I'll say it, contributing to open source feels intimidating.

Logically, in my head, I can put myself in the shoes of someone who is maintaining a project. And, while in that mindset, it's pretty easy to imagine that any help is better than no help. And, hell, if someone actually wants to work on something that I'm maintaining in order to improve its quality, add a feature, or simply make it easier to use and understand?!?!

That sounds amazing. Yes please. I'll take two.

But then I float back into my own head.

Trying to find a project to contribute to for open source is like wandering an art show thinking, "I could improve on that."

Now, here's where it gets tricky.

Sometimes the art show I'm wandering through is the kindergarten display the day after everyone learned finger painting. In that case, yes, I feel fairly confident that I can contribute something of value ("Jimmy, don't be jealous that my finger-painted house is a 3-D cube instead of a slightly slanted square with windows which somehow look straight through the house").

Yet, other times, it turns out that nice Monet fellow made those haystacks appear a bit hazy on purpose. And no, I really don't have anything to contribute there.

Finding the sweet spot, then, is imperative.

I had to look for something that I was familiar with, yet hadn't already been combed over by the best programming minds in the community, yet also felt worthwhile to contribute to. I mean, the whole point of this is to make a pull request worth merging, because, well, it's code worth using more than once.

Having found a project to contribute to, I agonized over the code and maybe wrote less than 50 lines in total. Yet it took me forever. Because open source code is public code. And I wanted to make sure it was correct, more than correct, worthy of sharing.

It was a slightly selfish endeavor as well.

After all, I started the quest to contribute to open source by asking the question: Which tools do I find myself reaching for time and again, only to have to rewrite the darn thing from scratch each time?

So, to sum up, contributing to open source is intimidating, time-consuming, and selfish.

And I should be doing more of it. Because open source code forms the foundation of the projects I work on every single day. If I'm not willing to chip in and maintain them, who should?

And, hey, any help is better than no help.
